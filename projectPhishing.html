The Challenge

Phishing attacks are evolving, bypassing traditional blacklists and content-based filters. The challenge lies in creating a system that not only detects these complex threats but does so in milliseconds before a user compromises their credentials.

We utilized a perfectly balanced dataset of 11,430 URLs (50% legitimate, 50% phishing) to train our models without bias.

87 Extracted Features

Analyzed attributes from the address bar, abnormal actions, and HTML/JS source code.

Feature Selection

Applied InfoGain and ReliefF filters to reduce dimensionality and isolate the most critical indicators.

Execution Pipeline

1

Data Aggregation

Legitimate web pages were sourced from Common Crawl, while active phishing instances were securely gathered from Phish-Tank and Open-Phish.

2

Algorithm Selection

Deployed three leading algorithms via the Weka toolkit: J48 (Decision Tree) for logical splitting, Random Forest (RF) for ensemble learning, and Multilayer Perceptron (MLP) for neural network correlation.

3

Optimization & Cross-Validation

Reduced the 48 features down to the Top 5, 10, and 15 using InfoGain. Validated the models strictly using a 10-fold cross-validation technique to ensure zero variance or overfitting.

4

The Verdict

While Random Forest achieved the highest raw accuracy (98.37%), combining the J48 Algorithm with InfoGain (Top 5 features) proved optimal for real-time web deployment, achieving a remarkable 95.31% accuracy in just 0.12 seconds.
